# 2. 数位 DP 的核心思想

## 2.1 什么是「数位」

把一个数按位拆开，例如 $324 = 3\cdot100 + 2\cdot10 + 4\cdot1$。数位 DP 就是**从高位到低位逐位填数字**，在这个过程中用 DP 统计满足条件的数的个数（或某种权值之和）。

## 2.2 为什么要逐位填 · 交互演示

因为「不超过 N」这个约束，在逐位填的时候有一个非常优美的结构：前面每一位都和 N 的对应位相等时，当前位受到 N 的限制；一旦某一位填了小于 N 对应位的数，后面所有位就自由了。下面的演示展示这个逐位填数的过程，以及 `limit`、`lead0` 状态如何随选择动态变化。

## 2.3 两个关键标志位

- **limit（上界限制）：** 本质是「前面所有位是否完全等于 N 的对应位」。limit=1 时当前位只能填 `0 ~ a[pos]`；一旦某位填小于 N 对应位的数，limit 变 0，之后可填 `0 ~ 9`。红色徽章表示受限。
- **lead0（前导零）：** 前面是否全是 0。它影响「数字有效长度」「某数字是否真正出现」「0 是否算合法位」等条件。黄色徽章表示有前导零。
- 两个都关掉时，状态是「自由的」，绿色徽章 —— 这种状态**才值得记忆化**。

> **WARN 易错点：** `limit=1` 或 `lead0=1` 的状态**不记忆化**。原因不是「缓存会出错」，而是它们**没有重复子问题**：limit=1 表示前缀被 N 唯一锁定，沿一条单链传播，每个状态只访问一次，缓存没有收益；更关键的是 limit=1 的状态依赖 a[pos] 的具体值，跨不同 N 复用会读到错误结果。详见模板章节的递归树演示。

## 2.4 整体流程

对问题「求 `[0, N]` 中满足条件 P 的数个数」：

1. 把 N 拆位存入数组（代码里通常 `a[len++]=N%10`，即低位在前，从 a[len-1] 最高位开始填）。
2. 写一个 `dfs(pos, limit, lead0, ...)`，`...` 是题目相关的额外状态，从 `pos=len-1` 递减到 `pos=-1`。
3. 在 `pos == -1` 时根据额外状态返回边界值（计数题通常是 1）。
4. 枚举当前位填什么数字，递归累加，带记忆化。
5. 最终答案 = `solve(R) - solve(L-1)`。
