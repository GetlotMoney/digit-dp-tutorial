# 前缀和与差分

**前缀和与差分**是竞赛中最基础、最常用的预处理技巧之一。核心思想非常朴素：把「区间求和」的多次查询通过一次 $O(n)$ 预处理变成 $O(1)$ 回答，把「区间加值」的多次操作通过差分数组压缩成单点操作后再统一还原。

适用场景：
- 多次询问数组某个区间的和 / 最值（前缀和 / 前缀最值）
- 对数组某个区间统一加一个值，最后求修改后的数组（差分）
- 二维矩阵的子矩阵求和 / 子矩阵加值

难度入门但威力巨大——很多看似复杂的题，拆开来看就是「在差分数组上做文章」。

---

## 核心思想

### 一维前缀和

给定数组 $a[1], a[2], \dots, a[n]$，定义前缀和数组：

$$
s[i] = \sum_{k=1}^{i} a[k], \quad s[0] = 0
$$

那么区间 $[l, r]$ 的和就可以 $O(1)$ 回答：

$$
\sum_{k=l}^{r} a[k] = s[r] - s[l-1]
$$

为什么 $s[0]=0$？因为当 $l=1$ 时，$s[r] - s[0] = s[r]$，恰好就是从第一个元素加到第 $r$ 个，不需要特判。这是竞赛中非常常见的「哨兵」技巧。

### 二维前缀和

把一维推广到矩阵。定义：

$$
s[i][j] = \sum_{x=1}^{i}\sum_{y=1}^{j} a[x][y]
$$

左上角 $(1,1)$、右下角 $(i,j)$ 的子矩阵和，同样 $O(1)$ 回答：

$$
\text{sum}(r_1,c_1,r_2,c_2) = s[r_2][c_2] - s[r_1-1][c_2] - s[r_2][c_1-1] + s[r_1-1][c_1-1]
$$

这里的「减两个再加回来一个」就是容斥原理。想象用四个角拼矩形：大矩形减去上面多余、左边多余，但左上角被减了两次，需要补回。

### 一维差分

差分是前缀和的逆运算。定义差分数组：

$$
d[i] = a[i] - a[i-1], \quad d[1] = a[1]
$$

差分数组有一个极其重要的性质：**对原数组 $[l, r]$ 区间内所有元素加 $v$，等价于 $d[l] \gets d[l]+v$，$d[r+1] \gets d[r+1]-v$**，只需两次 $O(1)$ 的单点操作。

为什么？因为差分数组的前缀和就是原数组本身。$d[l]+v$ 使得从位置 $l$ 开始的所有前缀和都多了 $v$；$d[r+1]-v$ 使得从位置 $r+1$ 开始的前缀和「消」回原来的值。

所有区间加操作完成后，对差分数组求一次前缀和即可得到最终数组。

### 二维差分

把一维差分推广到矩阵。对子矩阵 $(r_1,c_1) \sim (r_2,c_2)$ 内所有元素加 $v$，等价于在差分矩阵上做四次单点操作：

$$
d[r_1][c_1] \mathrel{+}= v, \quad d[r_1][c_2+1] \mathrel{-}= v, \quad d[r_2+1][c_1] \mathrel{-}= v, \quad d[r_2+1][c_2+1] \mathrel{+}= v
$$

和二维前缀和的容斥完全对称——因为二维差分就是二维前缀和的逆运算。所有操作完成后，对差分矩阵做一次二维前缀和即可还原。

## 模板代码

### 一维前缀和

```cpp
// 一维前缀和模板
// a[1..n] 为原数组（下标从 1 开始，方便处理边界）
// s[i] 表示 a[1] + a[2] + ... + a[i]
int n, a[N], s[N];

void build() {
    s[0] = 0;
    for (int i = 1; i <= n; i++) {
        s[i] = s[i - 1] + a[i]; // 递推：当前前缀和 = 上一个 + 当前值
    }
}

// 查询区间 [l, r] 的和，O(1)
int query(int l, int r) {
    return s[r] - s[l - 1];
}
```

### 二维前缀和

```cpp
// 二维前缀和模板
// a[i][j] 为原矩阵，s[i][j] 为二维前缀和
int n, m, a[N][M], s[N][M];

void build() {
    // 清零第 0 行和第 0 列（哨兵）
    for (int i = 0; i <= n; i++) s[i][0] = 0;
    for (int j = 0; j <= m; j++) s[0][j] = 0;
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            // 容斥原理：左上子矩阵和 + 当前行前缀 + 当前列前缀 - 重复部分 + 当前值
            s[i][j] = s[i - 1][j] + s[i][j - 1] - s[i - 1][j - 1] + a[i][j];
        }
    }
}

// 查询子矩阵 (r1,c1) 到 (r2,c2) 的和
int query(int r1, int c1, int r2, int c2) {
    return s[r2][c2] - s[r1 - 1][c2] - s[r2][c1 - 1] + s[r1 - 1][c1 - 1];
}
```

### 一维差分

```cpp
// 一维差分模板
// d[i] = a[i] - a[i-1]，d[0] = a[0]（视定义而定）
// 对原数组 [l, r] 区间加 v
int n, d[N];

void range_add(int l, int r, int v) {
    d[l] += v;       // 从 l 开始，所有前缀和都 +v
    d[r + 1] -= v;   // 从 r+1 开始，前缀和恢复正常
}

// 从差分数组还原原数组（对 d 求前缀和）
void restore() {
    for (int i = 1; i <= n; i++) {
        d[i] += d[i - 1]; // 还原后的 d[i] 就是最终的 a[i]
    }
}
```

### 二维差分

```cpp
// 二维差分模板
// 对子矩阵 (r1,c1) ~ (r2,c2) 所有元素加 v
int n, m, d[N][M];

void range_add(int r1, int c1, int r2, int c2, int v) {
    d[r1][c1]         += v;
    d[r1][c2 + 1]     -= v;
    d[r2 + 1][c1]     -= v;
    d[r2 + 1][c2 + 1] += v; // 容斥补回
}

// 从二维差分还原（对 d 做二维前缀和）
void restore() {
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= m; j++) {
            d[i][j] += d[i - 1][j] + d[i][j - 1] - d[i - 1][j - 1];
        }
    }
}
```

## 例题详解

### 例题 1：最大子数组和（LeetCode 53）

**题意：** 给定整数数组 $a[0..n-1]$，找一个连续子数组使得和最大，返回这个最大和。

**思路：** 维护前缀和的最小值 `min_s`。枚举右端点 $r$，当前位置的最优答案为 $s[r] - \min_{k<r} s[k]$，即「当前位置前缀和减去之前最小的前缀和」。$O(n)$ 一次扫描即可。

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int n = nums.size();
        int s = 0;        // 当前前缀和
        int min_s = 0;    // 之前的最小前缀和（s[0] = 0 为哨兵）
        int ans = INT_MIN; // 答案初始化为负无穷
        for (int i = 0; i < n; i++) {
            s += nums[i];                    // 累加前缀和
            ans = max(ans, s - min_s);       // 用「当前前缀和 - 最小前缀和」更新答案
            min_s = min(min_s, s);           // 更新最小前缀和
        }
        return ans;
    }
};
```

这里 $s - \min_s$ 本质上就是「枚举左端点 $l$，求 $s[r] - s[l-1]$ 中的最大值」，而 $\min_s$ 已经替我们记住了最优的 $l$。

### 例题 2：二维区域和检索（LeetCode 304）

**题意：** 给定二维矩阵，多次查询子矩阵的元素之和。

**思路：** 预处理二维前缀和 $s[i][j]$，查询 $O(1)$。

```cpp
class NumMatrix {
    vector<vector<int>> s; // 二维前缀和数组
public:
    NumMatrix(vector<vector<int>>& matrix) {
        int n = matrix.size(), m = matrix[0].size();
        // 多开一行一列作哨兵，避免边界判断
        s.assign(n + 1, vector<int>(m + 1, 0));
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= m; j++) {
                // 容斥原理
                s[i][j] = s[i-1][j] + s[i][j-1] - s[i-1][j-1] + matrix[i-1][j-1];
            }
        }
    }

    // 查询 (r1,c1) ~ (r2,c2) 的和
    int sumRegion(int r1, int c1, int r2, int c2) {
        r1++; c1++; r2++; c2++; // 转换为 s 数组的下标
        return s[r2][c2] - s[r1-1][c2] - s[r2][c1-1] + s[r1-1][c1-1];
    }
};
```

> **WARN 注意：** `matrix` 的下标从 0 开始，`s` 的下标从 1 开始（多开哨兵行列），查询时要 `+1` 转换，这一点在竞赛中非常容易写错。

## 常见错误

### 1. 前缀和数组越界

`s[l-1]` 在 $l=1$ 时会访问 $s[0]$，必须确保 $s[0]$ 已初始化为 0（哨兵）。很多段错误都是因为漏了这一步。

### 2. 二维前缀和容斥公式记混

口诀：**大减两小加一微**。即 $s[r_2][c_2] - s[r_1-1][c_2] - s[r_2][c_1-1] + s[r_1-1][c_1-1]$。记忆时想象用四块拼成一个矩形，减掉两块多余，再补回被减多的一块。如果实在记不住，从一维版本推导也可以。

### 3. 差分数组忘记还原

差分只是一种中间表示，最终需要**对差分数组求前缀和**才能得到修改后的原数组。忘记这一步直接输出差分数组会导致答案完全错误。

### 4. 差分的 $r+1$ 越界

区间 $[l, r]$ 差分操作需要修改 $d[r+1]$，当 $r=n$ 时 $d[n+1]$ 越界。解决方法：数组多开一个位置，或者在操作前特判 $r < n$。

### 5. 混淆前缀和与差分的方向

前缀和是从「区间信息」推「单点信息」（$s[i] = s[i-1] + a[i]$），差分是反过来。在一道题中既要用前缀和又要用差分时，容易搞混哪个数组正在操作。建议用不同变量名区分（如 `ps` 和 `df`），不要复用同一数组。
