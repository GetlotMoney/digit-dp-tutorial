# 区间DP

区间DP是一类以「区间」为状态维度的动态规划。核心思路是：**将一个大区间拆成两段小区间，枚举分割点，合并两段的最优解**。它适用于求解「在一个序列上按某种方式合并/删除/匹配，使总代价最小/最大」的问题。

典型应用包括：石子合并（最小合并代价）、矩阵链乘法（最少乘法次数）、戳气球（最大得分）、回文串分割等。

**何时使用区间DP？** 当问题满足以下特征：
- 输入是一个序列（数组/字符串），且问题关心的是**子序列/子区间**的最优值
- 可以把一个区间 $[l, r]$ 的答案拆成两个子区间 $[l, k]$ 和 $[k+1, r]$ 的答案之和（或某种合并）
- 区间长度从小到大递推，短区间的结果用来拼接长区间

## 核心思想

### 状态定义

令 $dp[l][r]$ 表示区间 $[l, r]$（闭区间）上的最优解。

### 转移方程

枚举分割点 $k$（$l \le k < r$），将区间 $[l, r]$ 拆为 $[l, k]$ 和 $[k+1, r]$：

$$dp[l][r] = \min_{l \le k < r} \{ dp[l][k] + dp[k+1][r] + cost(l, r, k) \}$$

其中 $cost(l, r, k)$ 是合并两段时的额外代价，具体含义因题而异。对于求最大值的问题，把 $\min$ 换成 $\max$ 即可。

### 枚举顺序

关键点：**区间长度从小到大枚举**。因为转移时需要短区间的结果，必须保证算 $dp[l][r]$ 时，所有更短的区间已经算好了。

```
长度 len = 1:  [0,0] [1,1] [2,2] ...
长度 len = 2:  [0,1] [1,2] [2,3] ...
长度 len = 3:  [0,2] [1,3] [2,4] ...
...
长度 len = n:  [0,n-1]
```

### 经典问题：石子合并（直线版）

有 $n$ 堆石子排成一排，每次可以合并**相邻**的两堆，合并代价为两堆石子数之和。求合并成一堆的最小总代价。

- $dp[l][r]$：合并 $[l, r]$ 内所有石子为一堆的最小代价
- 前缀和 $s[i] = a[1] + a[2] + \cdots + a[i]$，区间和用 $s[r] - s[l-1]$ 快速计算
- 转移：枚举最后一次合并在哪（分割点 $k$）

$$dp[l][r] = \min_{l \le k < r} \{ dp[l][k] + dp[k+1][r] \} + (s[r] - s[l-1])$$

> 最后加的 $(s[r] - s[l-1])$ 是把两堆合成一堆的代价，等于这一段所有石子的总重量。

边界条件：$dp[i][i] = 0$（一堆石子不需要合并）。

### 环形区间DP

如果石子排成一个**环**（即第一堆和最后一堆相邻），怎么处理？

**技巧：破环成链**。把长度为 $n$ 的数组复制一份拼到后面，变成长度为 $2n$ 的数组。然后在这个数组上跑区间DP，最终答案为：

$$ans = \min_{i=1}^{n} dp[i][i+n-1]$$

即在所有长度为 $n$ 的区间中取最优解。这个技巧把环形问题退化成了直线问题。

### 矩阵链乘法

给定 $n$ 个矩阵，第 $i$ 个矩阵的维度为 $p_{i-1} \times p_i$。矩阵乘法满足结合律但不同括号方案的乘法次数不同，求最少乘法次数。

- $dp[l][r]$：计算 $A_l \times A_{l+1} \times \cdots \times A_r$ 的最少乘法次数
- 转移：枚举最后一步乘法的分割点 $k$

$$dp[l][r] = \min_{l \le k < r} \{ dp[l][k] + dp[k+1][r] + p_{l-1} \cdot p_k \cdot p_r \}$$

边界条件：$dp[i][i] = 0$（一个矩阵不需要乘法）。

## 模板代码

下面是区间DP的通用模板，以「石子合并（直线版，求最小值）」为例。洛谷 P1880。

```cpp
#include <bits/stdc++.h>
using namespace std;
const int N = 210;
const int INF = 0x3f3f3f3f;

int n;
int a[N], s[N];        // a[i] 第 i 堆石子数，s[i] 前缀和
int dp[N][N];          // dp[l][r] 合并 [l,r] 的最小代价

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        s[i] = s[i - 1] + a[i];  // 前缀和，用于快速求区间和
    }

    // 初始化：区间长度为 1 时，合并代价为 0
    for (int i = 1; i <= n; i++) dp[i][i] = 0;

    // 按区间长度从小到大枚举
    for (int len = 2; len <= n; len++) {          // len = 区间长度
        for (int l = 1; l + len - 1 <= n; l++) {  // l = 左端点
            int r = l + len - 1;                   // r = 右端点
            dp[l][r] = INF;
            // 枚举分割点 k，将 [l,r] 拆为 [l,k] 和 [k+1,r]
            for (int k = l; k < r; k++) {
                dp[l][r] = min(dp[l][r],
                    dp[l][k] + dp[k + 1][r] + s[r] - s[l - 1]);
            }
        }
    }

    cout << dp[1][n] << endl;
    return 0;
}
```

> **TIP 模板要点：** 三重循环的结构是固定的 —— 外层枚举区间长度，中层枚举左端点，内层枚举分割点。只需修改 `cost` 部分（这里是 `s[r] - s[l-1]`）和 `min`/`max` 选择，就能适配不同题目。

## 例题详解

### 例题1：石子合并（环形）—— 洛谷 P1880

**题意：** $n$ 堆石子围成一圈，每次合并相邻两堆，代价为两堆石子数之和。分别求合并成一堆的**最小**和**最大**总代价。

**思路：** 破环成链，把数组复制一份拼到后面，然后对长度 $2n$ 的数组跑区间DP。

```cpp
#include <bits/stdc++.h>
using namespace std;
const int N = 410;
const int INF = 0x3f3f3f3f;

int n;
int a[N], s[N];
int dp_min[N][N], dp_max[N][N];

int main() {
    cin >> n;
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        a[i + n] = a[i];           // 破环成链：复制一份拼在后面
    }
    for (int i = 1; i <= 2 * n; i++)
        s[i] = s[i - 1] + a[i];

    for (int i = 1; i <= 2 * n; i++)
        dp_min[i][i] = dp_max[i][i] = 0;

    for (int len = 2; len <= n; len++) {
        for (int l = 1; l + len - 1 <= 2 * n; l++) {
            int r = l + len - 1;
            dp_min[l][r] = INF;
            dp_max[l][r] = 0;
            for (int k = l; k < r; k++) {
                dp_min[l][r] = min(dp_min[l][r],
                    dp_min[l][k] + dp_min[k + 1][r] + s[r] - s[l - 1]);
                dp_max[l][r] = max(dp_max[l][r],
                    dp_max[l][k] + dp_max[k + 1][r] + s[r] - s[l - 1]);
            }
        }
    }

    int ans_min = INF, ans_max = 0;
    for (int i = 1; i <= n; i++) {
        ans_min = min(ans_min, dp_min[i][i + n - 1]);
        ans_max = max(ans_max, dp_max[i][i + n - 1]);
    }
    cout << ans_min << "\n" << ans_max << endl;
    return 0;
}
```

### 例题2：戳气球 —— LeetCode 312

**题意：** 有 $n$ 个气球，每个气球有一个分数 $val[i]$。戳破第 $i$ 个气球获得 $val[left] \times val[i] \times val[right]$ 分，其中 $left$ 和 $right$ 是 $i$ 左右**未被戳破**的气球。求最大总分。

**思路：** 正向思维（每次戳哪个）很难做区间DP，因为戳完之后左右邻居会变化。**逆向思维**：考虑最后一个被戳破的气球。设 $dp[l][r]$ 表示开区间 $(l, r)$ 内的气球全部被戳破的最大得分（$l$ 和 $r$ 本身不戳），则：

$$dp[l][r] = \max_{l < k < r} \{ dp[l][k] + dp[k][r] + val[l] \cdot val[k] \cdot val[r] \}$$

在 $val$ 数组两端各加一个值为 1 的虚拟气球，答案就是 $dp[0][n+1]$。

```cpp
#include <bits/stdc++.h>
using namespace std;

int dp[510][510];  // dp[l][r] 开区间 (l,r) 内全戳掉的最大分

int maxCoins(vector<int>& nums) {
    int n = nums.size();
    // 两端加虚拟气球
    vector<int> val(n + 2);
    val[0] = val[n + 1] = 1;
    for (int i = 1; i <= n; i++) val[i] = nums[i - 1];

    // 按区间长度枚举
    for (int len = 2; len <= n + 1; len++) {        // len = r - l
        for (int l = 0; l + len <= n + 1; l++) {
            int r = l + len;
            for (int k = l + 1; k < r; k++) {
                // k 是最后被戳破的气球
                dp[l][r] = max(dp[l][r],
                    dp[l][k] + dp[k][r] + val[l] * val[k] * val[r]);
            }
        }
    }
    return dp[0][n + 1];
}
```

> **WARN 注意：** 这题的 $dp[l][r]$ 是**开区间** $(l, r)$，和石子合并的闭区间 $[l, r]$ 不同。做题时务必看清区间的开闭定义，这是最容易出错的地方。

## 常见错误

### 1. 枚举顺序搞反

**错误：** 外层枚举左端点，内层枚举长度。这会导致算 $dp[l][r]$ 时，需要的短区间结果还没算出来。

**正确：** 外层枚举长度 `len`，中层枚举左端点 `l`，内层枚举分割点 `k`。这是固定的三重循环结构。

### 2. 分割点范围写错

**错误：** 写成 `k = l` 到 `k <= r`，导致区间拆分时出现空区间或越界。

**正确：** 分割点 `k` 的范围是 `l <= k < r`，保证左半段 $[l, k]$ 和右半段 $[k+1, r]$ 都非空。

### 3. 开闭区间混淆

石子合并用的是**闭区间** $dp[l][r]$（$l$ 和 $r$ 都包含），戳气球用的是**开区间** $dp[l][r]$（$l$ 和 $r$ 都不包含）。转移方程、边界条件、循环写法都会因此不同。做题前先确认区间的开闭定义。

### 4. 环形处理遗漏

**错误：** 环形题目直接用长度 $n$ 而不是 $2n$，或者最后没有在所有长度为 $n$ 的区间中取最优。

**正确：** 复制数组拼接到长度 $2n$，最终答案为 $\min_{i=1}^{n} dp[i][i+n-1]$（或 $\max$）。

### 5. 初始化不完整

**错误：** 只初始化了 $dp[i][i] = 0$，但忘记把 $dp[l][r]$（$l \neq r$）初始化为 INF（求最小值）或 0（求最大值）。

**正确：** 在每一轮转移前，先给 $dp[l][r]$ 赋初值。如果用全局数组且求最小值，确保全局初始化为 INF。
