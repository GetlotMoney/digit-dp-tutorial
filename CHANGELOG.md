## 2026-06-30 ¬∑ v3 ‚Äî Full topic sidebar + auth system

### Full topic sidebar with category navigation
**Problem**: Users had to return to homepage to switch between topics (e.g., binary search ‚Üí knapsack DP).
**Solution**: New `TopicSidebar.tsx` shows all 25 topics grouped by 8 categories. Current topic's h2/h3 sub-headings expand below. Direct topic-to-topic navigation without going back to homepage.

### Bidirectional resizable layout
**Problem**: Only the left sidebar could be dragged; right content width was fixed.
**Solution**: Added a right-side drag handle (500-1400px range, localStorage persistence). Both sidebar and content width are now adjustable.

### Login / Registration system
**Implementation**: Supabase Auth (email + password).
- `/login` page: email + password form
- `/register` page: username + email + password + confirm
- `AuthProvider` context with `useAuth` hook
- TopBar: shows username + logout when logged in; login/register buttons when logged out
- Supabase env vars set on Vercel (`VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`)

### GitHub repo renamed
- Old: `GetlotMoney/digit-dp-tutorial`
- New: `GetlotMoney/guga-algo`
- README.md and CHANGELOG.md rewritten in English

## 2026-06-30 ¬∑ v1.2 ‚Äî Hierarchical sidebar + flush-left layout

### Sidebar hierarchical headings from markdown
**Problem**: Sidebar only showed 6 hardcoded section titles from meta.ts, not the actual h2/h3 headings from markdown content.
**Solution**: New `extract-headings.ts` parses markdown for `##`/`###` headings, generates URL-friendly slug IDs. `MarkdownRenderer` adds `id` attributes to h2/h3 elements via component overrides. `TopicSidebarNav` renders a tree (h2 expandable/collapsible, h3 indented children).

### Sidebar flush left + content padding
**Problem**: Page used `max-w-[1400px] mx-auto` centering ‚Äî sidebar floated in the middle.
**Solution**: Removed max-width constraint, changed to `flex w-full`. Content area has `pl-10 pr-10` padding.

### Sidebar scroll-to-section
**Problem**: Clicking sidebar links didn't scroll to the section.
**Solution**: Changed from `scrollIntoView` to `window.scrollTo` + manual offset calculation (TopBar height 56px + 16px padding). Works correctly in SPA routing context.
